# Creator Patterns Guide

This guide turns common Roblox growth advice into practical DPFTS rules. It is not a promise that copying a top game will make a hit. Big Roblox experiences win because their loop is obvious, fast, social, measurable, and updated. The boring fundamentals matter more than whatever trend is loud this week.

## What Strong Roblox Experiences Usually Do

- Start the player near the core loop.
- Teach by action before text.
- Give a small win quickly.
- Show a visible long-term goal.
- Make other players readable and useful.
- Keep UI tap-friendly on mobile.
- Track where players quit.
- Iterate based on retention, completion, errors, and feedback.
- Add content on a rhythm players can understand.

## What Weak AI-Generated Experiences Usually Do

- Spawn the player in a big empty area.
- Explain the whole game in a wall of text.
- Add 12 buttons before one fun action.
- Hide the main loop behind menus.
- Monetize before proving fun.
- Add quests that are just chores.
- Add daily rewards with no reason to play after claiming.
- Add social features that do not affect gameplay.
- Chase trends without a playable core.

## Big-Creator Pattern: Clear Core Loop

Top Roblox games usually make the loop painfully clear:

1. Do the main action.
2. Get feedback.
3. Earn progress.
4. Unlock or upgrade.
5. Repeat with a better target.

DPFTS should reject systems that add progression without a loop. A quest, shop, pet, weapon, or hub only works if it pushes the player back into a repeatable action.

## Big-Creator Pattern: Fast First Session

Roblox's own guidance centers retention around FTUE and core-loop completion. A good first session should:

- Put the player in control quickly.
- Show one clear objective.
- Let the player complete the core loop once.
- Give a joyful first reward.
- Preview what comes next.

Do not start with lore, loading theatrics, or a full economy tutorial.

## Big-Creator Pattern: Social Readability

Strong Roblox games often let players understand what others are doing by watching:

- Other players show progress through pets, tools, titles, trails, builds, or zones.
- Hubs route players past each other.
- Trading, parties, races, rounds, co-op goals, and leaderboards create reasons to notice people.
- Social features support the core loop instead of sitting in a separate menu.

Bad social design is a giant plaza where nothing meaningful happens.

## Big-Creator Pattern: Live Updates Without Chaos

Good live updates:

- Add to the existing loop.
- Give returning players a clear reason to check in.
- Use events, limited items, new zones, or new quests honestly.
- Keep old players from running out of goals.
- Avoid breaking the first-time experience.

Bad live updates:

- Add random systems because a competitor has them.
- Move the spawn every week.
- Add paid items before balance is stable.
- Make returning players read patch notes before they can play.

## How To Use Analytics

DPFTS should suggest simple tracking questions:

- Where do new players quit?
- Do players complete the first core loop?
- Which tutorial step loses the most players?
- Do players return after one day?
- Are players stuck before the first reward?
- Which quests are completed, ignored, or abandoned?
- Does a new UI reduce purchase confusion or increase misclicks?

Use metrics to find friction. Do not use metrics as an excuse to make dishonest design.

## Better To Do

- Build a small loop and test it early.
- Greybox the route before decorating.
- Use one primary objective in the first minute.
- Add one useful social signal.
- Add rewards that push players back into play.
- Measure tutorial and core-loop completion.
- Update with purpose.

## Better Not To Do

- Do not copy a simulator template without a twist.
- Do not add five currencies in the first screen.
- Do not hide gameplay behind daily rewards.
- Do not make quests that only count time spent walking.
- Do not add admin spectacle that ruins normal players' progress.
- Do not use fake urgency, fake Robux claims, or misleading thumbnails.
- Do not buy traffic before retention is healthy.
